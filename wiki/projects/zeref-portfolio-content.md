# Zeref Portfolio and Content Strategy

How Zeref gets packaged for external visibility — portfolio, GitHub, LinkedIn, Substack.

## Portfolio Positioning

Zeref is portfolio-worthy because it demonstrates **systems-level thinking** across:
- Product (problem definition, strategy, UX flows)
- Design (register model, accessibility, UI system)
- Technical (5-layer architecture, modular skill fleet, CLI plugin)
- Documentation (wiki memory layer, decision log, QA audit)

**Angle:** Zeref is not "I built a ChatGPT wrapper." It is "I designed an execution OS with compounding memory, deterministic routing, and a 112-skill modular fleet."

## Content Goals (LinkedIn / Substack)

| Goal | Angle |
|------|-------|
| Explain the problem | AI treats every session as disposable conversation instead of compounding execution |
| Show the solution | Zeref creates compounding memory — each session builds on the last |
| Prove systems thinking | The architecture itself (5 layers, routing model, wiki) is the proof of work |
