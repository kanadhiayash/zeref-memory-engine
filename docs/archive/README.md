# Zeref Public Archive

This folder keeps public-safe legacy context that should not sit on the main README.

Archive rules:

- Keep public-safe history here.
- Do not store secrets, private paths, private user data, credentials, or unverified claims.
- Do not use archive files as current product truth.
- Current product truth lives in `README.md`, `AGENTS.md`, and current docs.
- Old benchmark claims must stay clearly marked as historical.
- Old naming references should be marked as legacy.

## Archive pages

- `INSPIRATIONS.md` records public inspiration and lineage links.
- `PUBLIC_SURFACE_NOTES.md` records what was removed from the README and why.
