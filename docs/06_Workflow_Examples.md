## Workflow 2: Firebase Auth Build

**Scenario:** Building email + Google OAuth login for a mobile app. Need architecture, implementation, and security review.

### Step 1 — Architecture first

**Input:**
```
Design the auth system for a React Native app. Email + Google OAuth. Firebase backend.
```

**Routing:**
```
Lead: zeref-dev-solution-architect
Support: zeref-dev-firebase-specialist, zeref-dev-security-engineer
QA: (deferred to implementation)
```

**Output:**
- Auth flow diagram (states: unauthenticated → verified → active session)
- Firebase service choices (Auth + Firestore for user profiles)
- Security decisions: token storage, refresh strategy, session timeout
- Risk flags: account linking edge cases, offline behavior

### Step 2 — Build the auth module

**Input:**
```
Write the Firebase auth implementation. Email/password + Google. React Native. Include error handling.
```

**Routing:**
```
Lead: zeref-dev-firebase-specialist
Support: zeref-dev-frontend-engineer
QA: zeref-qa-security-tester
```

**Output:**
```javascript
// auth.js — Firebase Auth wrapper
// Generated by Zeref OS | zeref-dev-firebase-specialist

import auth from '@react-native-firebase/auth';
import { GoogleSignin } from '@react-native-google-signin/google-signin';

export async function signInWithEmail(email, password) {
  try {
    const userCredential = await auth().signInWithEmailAndPassword(email, password);
    return { user: userCredential.user, error: null };
  } catch (error) {
    return { user: null, error: mapAuthError(error.code) };
  }
}

export async function signInWithGoogle() {
  try {
    await GoogleSignin.hasPlayServices();
    const { idToken } = await GoogleSignin.signIn();
    const googleCredential = auth.GoogleAuthProvider.credential(idToken);
    const userCredential = await auth().signInWithCredential(googleCredential);
    return { user: userCredential.user, error: null };
  } catch (error) {
    return { user: null, error: mapAuthError(error.code) };
  }
}

function mapAuthError(code) {
  const errors = {
    'auth/user-not-found': 'No account found with this email.',
    'auth/wrong-password': 'Incorrect password.',
    'auth/email-already-in-use': 'An account already exists with this email.',
    'auth/network-request-failed': 'No internet connection.',
  };
  return errors[code] || 'Authentication failed. Please try again.';
}
```

**Security QA check:**
- ✅ No tokens stored in AsyncStorage (memory only)
- ✅ Error messages don't expose internals
- ✅ Google Sign-In uses `hasPlayServices()` check
- ⚠️ Risk: No rate limiting on email auth — add Firebase App Check

## Workflow 4: Business Validation

**Scenario:** Idea for a B2B SaaS tool. Need to know if it's worth building before spending 3 months on it.

### Step 1 — Market check

**Input:**
```
Is there a market for an AI workflow automation tool targeting Toronto-based startups under 50 people?
```

**Routing:**
```
Lead: zeref-biz-business-validator
Support: zeref-biz-market-research-analyst, zeref-biz-competitive-intelligence-analyst
QA: zeref-qa-rubric-alignment-auditor
```

**Output structure:**
- Market size estimate (TAM/SAM/SOM — labeled as estimates, not research)
- Existing competitors and their positioning gaps
- Key assumptions that must be true for this to work
- Validation experiments ranked by speed and cost
- Go/no-go signal based on assumptions

### Step 2 — Validation plan

**Input:**
```
Give me a 2-week validation plan I can run without writing any code.
```

**Routing:**
```
Lead: zeref-biz-business-validator
Support: zeref-biz-product-market-fit-analyst
```

**Output:**
- Week 1: 10 customer interviews (script included)
- Week 2: Landing page test (copy provided) + 5 warm intros
- Success criteria: 3+ people say "when can I use this?"
- Failure signal: Everyone says "interesting" but won't give email


```
"What decisions came out of my last meeting about the app redesign? Check email."

```
