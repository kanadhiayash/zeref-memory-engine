# Public Surface Policy

The public surface is everything a visitor can read before cloning or installing Zeref.

## Primary surfaces

- `README.md`
- `SECURITY.md`
- `CONTRIBUTING.md`
- `AGENTS.md`
- `docs/BENCHMARK_REPORT.md`
- `docs/TRUST_AUDIT.md`
- GitHub releases
- GitHub wiki

## Rules

1. Do not claim external benchmark leadership without reproducible external runs.
2. Do not claim production readiness without a named release gate.
3. Do not hard-code test counts unless generated by the release process.
4. Do not link to deleted tags.
5. Do not publish private paths, credentials, internal notes, or user-specific data.
6. Keep historical material in `CHANGELOG.md`.
7. Mark fixture-only benchmark adapters clearly.
8. Separate facts, assumptions, unknowns, and risks in public docs.
9. Keep the README concise.
10. Make security reporting private and obvious.

## Public claim levels

| Level | Allowed wording |
|---|---|
| Verified local | Deterministic local benchmark gate passed. |
| Fixture verified | Adapter fixture passed locally. |
| External verified | Named external dataset was run with reproducible commands. |
| Comparative | Named tools were compared with dated methodology. |
| Not allowed | Best, world top, 10/10, production-grade, or secure by default without proof. |
